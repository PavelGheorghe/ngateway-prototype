How DNS resolution works globally

From a hostname like domain.com to an IP address (typically an A or AAAA record): your device asks a recursive resolver, which either answers from cache or walks the DNS tree (root → TLD → authoritative) and returns the result.

Flow overview

  1. Stub resolver (OS) sends the query to a recursive resolver.
  2. On a cache miss, the resolver queries root, then .com TLD, then authoritative NS for domain.com.
  3. The authoritative server returns the A/AAAA record; the resolver caches it (TTL) and replies to the client.
